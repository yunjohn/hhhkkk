# 海阔小程序规则经验总结

## 1. 长口令处理

- 不要把完整口令直接 Base64 后复制到 Android 剪贴板，通常会更长。
- 官方推荐的短入口是订阅或云口令。
- 小程序订阅推荐格式：

```txt
海阔视界合集规则订阅￥home_sub￥https://raw.githubusercontent.com/owner/repo/main/sub.json
```

- `sub.json` 内容使用规则 JSON 数组：

```json
[
  {
    "title": "规则名",
    "find_rule": "js:\n..."
  }
]
```

- 单条云口令可用：

```txt
云口令，复制整条口令打开软件即可导入
https://raw.githubusercontent.com/owner/repo/main/rule.txt@import=js:fetch(input)
```

## 2. GitHub 订阅仓库结构

推荐结构：

```txt
sub.json       订阅源，数组，放所有小程序规则 JSON
youwu.json     单个规则 JSON，方便调试和备份
youwu.txt      完整口令，方便直接导入
README.md
```

如果网络导致 `git push` 失败，可以用 `gh api` 更新关键文件：

```powershell
$repo='owner/repo'
$sha = gh api repos/$repo/contents/sub.json --jq '.sha'
$content=[Convert]::ToBase64String([IO.File]::ReadAllBytes('D:\path\sub.json'))
$body=@{message='Update subscription'; content=$content; sha=$sha; branch='main'} | ConvertTo-Json -Compress
$body | gh api repos/$repo/contents/sub.json -X PUT --input -
```

订阅能否生效，关键只看远端 `sub.json`。

## 3. 图片站优先用 `pics://`

图集类站点优先使用：

```js
url: detailUrl + $('').lazyRule(() => {
  // 抓详情页和分页
  // 提取图片数组 arr
  return 'pics://' + arr.join('&&');
})
```

优点：

- 由海阔图片浏览协议处理图片流。
- 不需要自己构造二级 `pic_1_full` 列表。
- 对图集阅读体验更好。
- 口令通常更短。

常用并发抓分页：

```js
var urls = [{url: input}];
for (var k = 2; k <= pages; k++) {
  urls.push({url: input.split('?')[0] + '?page=' + k});
}
var arr = [];
var seen = {};
bf(urls).forEach(html => {
  extractImages(html).forEach(u => {
    if (u && !seen[u]) {
      seen[u] = 1;
      arr.push(u);
    }
  });
});
return 'pics://' + arr.join('&&');
```

## 4. 不要强行使用 `#autoPage#`

`#autoPage#` 适合小说章节类场景。

图集站如果父级列表是“下一套图”，直接加 `#autoPage#` 可能会自动加载下一套图，而不是当前图集下一页。

图集分页更适合：

- `pics://` 一次聚合全部图片。
- 或者分段聚合：当前页 + 后几页。
- 或者底部放“下一页/继续加载”按钮。

## 5. 图片地址处理经验

列表缩略图建议同时写：

```js
{
  pic_url: img,
  img: img
}
```

有些海阔布局只认其中一个字段。

不要随意给图片加 `@headers=`。如果浏览器/调试口令能显示直链缩略图，优先用纯 URL。

常见图片还原：

```js
function dec(u) {
  u = String(u || '').replace(/&amp;/g, '&');
  if (u.indexOf('redirect.php?url=') > -1) {
    u = decodeURIComponent(u.split('redirect.php?url=')[1]);
  }
  return u;
}
```

## 6. HTML 解析策略

优先用正则解析稳定块，避免选择器在异常 HTML 结构中失效。

常用属性读取函数：

```js
function attr(tag, name) {
  tag = String(tag || '');
  var s = tag.toLowerCase();
  var i = s.indexOf(name + '=');
  if (i < 0) return '';
  i += name.length + 1;
  var q = tag.charAt(i);
  var j = tag.indexOf(q, i + 1);
  return j > i ? tag.slice(i + 1, j) : '';
}
```

这个写法比动态 `RegExp` 更不容易因为口令字符串转义出错。

## 7. 变量兼容

不要假设所有环境都有：

```js
MY_CATE
MY_RULE
```

使用前要判断：

```js
typeof MY_PAGE == 'undefined' ? 1 : MY_PAGE
```

或者直接从 `MY_URL` 解析。

## 8. 调试方法

先做一个小调试规则测试缩略图字段和 URL：

```js
setHomeResult([
  {title:'直链', pic_url:u, img:u, url:u, col_type:'movie_2'}
]);
```

如果点击能打开但首页无缩略图，优先检查：

- 是否同时写了 `pic_url` 和 `img`。
- `col_type` 是否支持缩略图。
- URL 是否被 `@headers` 或代理格式干扰。

本地可用 Node 校验口令：

```js
const rule = JSON.parse(text.slice(marker.length));
new Function(rule.find_rule.replace(/^js:\n?/, ''));
new Function(rule.searchFind.replace(/^js:\n?/, ''));
```
